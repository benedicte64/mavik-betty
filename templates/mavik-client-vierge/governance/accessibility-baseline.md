# Socle accessibilité du programme vierge

## Références

- conception : WCAG 2.2 niveau AA ;
- référence européenne TIC : EN 301 549 v3.2.1, avec WCAG 2.1 AA pour le web ;
- méthode française publiée : RGAA 4.1.2, 106 critères et 13 thématiques ;
- veille : RGAA 5, annoncé pour fin 2026.

## Garanties non optionnelles

- aucun droit métier ne dépend d'une préférence d'accessibilité ;
- aucune information essentielle ne dépend uniquement de la vue, du son, de la couleur ou du mouvement ;
- les actions restent utilisables au clavier, avec un ordre de focus logique ;
- les changements dynamiques importants sont restitués par les technologies d'assistance ;
- ce que l'assistant dit existe aussi sous forme textuelle ;
- toute action importante est expliquée, confirmée, tracée et réversible lorsque cela est possible ;
- une option n'est jamais qualifiée d'accessible sur la seule base de tests automatiques.

## Promotion d'une option

Une option reste désactivée par défaut. Pour devenir éprouvée, elle doit disposer de tests automatiques, d'un parcours clavier, d'un essai lecteur d'écran, d'une vérification par rôle, d'un audit humain et d'un essai avec une personne concernée.
